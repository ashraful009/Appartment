# Project Overview
This project is a full-stack web application consisting of a React-based frontend (`client`) and a Node.js/Express backend (`server`). The frontend provides an interactive user interface constructed with React and structured to handle different user roles such as generic users, sellers, and administrators. The backend serves a REST API connected to a MongoDB database, handling core features including user authentication, data management, and media file uploads.

## Frontend (`/client`)
Built with React (v19) and bundled using Vite, leveraging Tailwind CSS for rapid styling and responsive design.

### Key Technologies
- **React 19 & Vite**
- **Tailwind CSS & PostCSS** (Styling)
- **React Router DOM** (Routing)
- **Axios** (HTTP Client)
- **Recharts** (Data Visualization)
- **Lucide React** (Icons)

### Directory Structure
- `client/src/components/` - Reusable UI components. Often organized by role/feature (e.g., `/admin`, `/seller`).
- `client/src/context/` - React Context providers for global state management (e.g., auth, themes).
- `client/src/pages/` - Top-level route components representing full views or pages in the application.
- `client/src/utils/` - Shared utility functions and common logic.
- `client/src/assets/` - Static assets including images and localized stylesheets.
- `client/package.json` - Frontend dependencies, configuration, and scripts.
- `client/vite.config.js` - Configuration for the Vite development server and builder.

## Backend (`/server`)
Built with Node.js and Express.js, providing a robust RESTful API that interfaces with a MongoDB database via Mongoose.

### Key Technologies
- **Node.js & Express.js**
- **MongoDB & Mongoose** (Database & ODM)
- **JWT & bcryptjs** (Authentication & Security)
- **Cloudinary & Multer** (Media Uploads and Storage)
- **CORS & dotenv** (Environment and Cross-Origin Configuration)

### Directory Structure
- `server/src/models/` - Mongoose schemas that define the structure of the database documents.
- `server/src/controllers/` - Route handlers containing the core business logic and database interactions.
- `server/src/routes/` - Express router definitions that map URLs/endpoints to the corresponding controller functions.
- `server/src/middleware/` - Express middlewares for tasks such as token verification, error handling, and file parsing.
- `server/src/config/` - Configuration setups, such as the initial database connection logic.
- `server/src/utils/` - Helper and generic utility functions used across various backend services.
- `server/index.js` - The main entry point that configures and starts the Express server.
- `server/package.json` - Backend dependencies and run scripts (e.g., `start`, `dev`).
